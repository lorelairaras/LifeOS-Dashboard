# Product Vision — LifeOS Portfolio Dashboard

**Version:** v0.1.0  
**Date:** 2026-06-03  
**Author:** Rory  
**Status:** Draft

---

## Vision Statement

LifeOS is the portfolio site that works as hard as the person it represents — publicly showing real craft to employers, and privately keeping that person's daily work organised.

It is not a template. It is a living system: one codebase, one repository, one deployed URL that grows with the developer's career and productivity needs.

---

## Why This Exists

Most developers build a portfolio once, deploy it, and forget it. It becomes a snapshot that quickly goes stale. LifeOS is designed to be actively maintained because the private dashboard gives it daily utility. Every time the task tracker is used, every time a job application is logged, every time a prompt is saved — the system stays alive and the portfolio stays current.

The public side demonstrates what the developer can build. The private side proves they actually use what they build.

---

## Core Values

**Useful before impressive.** The dashboard must solve real daily problems first. Visual effects and impressive tech come later, once the core is solid.

**Free by default.** Every tool and service used must have a viable free tier. The project must be reproducible by anyone with a GitHub account and no budget.

**Documentation as a skill.** The docs folder is not a formality. It is part of the portfolio. A developer who can write a clear project brief, a system architecture document, and a QA plan is demonstrating a skill that most developers skip.

**Build thin, then deep.** Ship the thinnest version that demonstrates the concept. Iterate. Never add complexity to the MVP that the user hasn't asked for.

**Honest quality.** Tests either pass or they don't. Deployment either works or it doesn't. No fabricated results. No "it should work." Only "it does work and here is the evidence."

---

## What Success Looks Like in 6 Months

- The public portfolio is live, accessible, and shared in job applications.
- The dashboard is used daily — tasks, prompts, job tracking, and budget entries are all in one place.
- The GitHub repository is public and shows consistent commit history, clean PRs, and maintained documentation.
- An employer who views the repository can understand the project, the decisions made, and the quality standards applied — without needing to ask questions.
- At least one case study documents the LifeOS project itself as a portfolio piece.

---

## What Success Looks Like in 12 Months

- Supabase persistence is live — data is saved across devices.
- Authentication protects the private dashboard.
- At least one AI-assisted feature is live (prompt improver, resume helper, or similar).
- A 3D or visually distinctive landing page differentiates the portfolio from standard templates.
- The project is referenced in interviews as a working example of product thinking, documentation, and technical delivery.

---

## What This Is Not

- Not a SaaS product for other users.
- Not a productivity tool with a public-facing sign-up.
- Not a blogging platform.
- Not a social network.
- Not a showcase for frameworks or libraries for their own sake.

---

## Strategic Bets

1. **Documentation quality as differentiator.** Most developers do not document their personal projects. A well-documented project signals product thinking and professional discipline.
2. **Daily utility drives maintenance.** A private dashboard used every day stays maintained. A purely public portfolio often does not.
3. **Free stack limits scope, which is a feature.** Constraints force decisions. Choosing the simplest tool that works demonstrates sound engineering judgement.
