## Log Format

Each test run is recorded as a block:

```

Phase: [phase]
Environment: local | Vercel
Branch: [branch name]
Tests run: [number]
Tests passed: [number]
Tests failed: [number]
Tests skipped: [number]
Notes: [observations]
```

Failed tests are linked to bug fix log entries: `BUG-[ID]` in `docs/qa/bug-fix-log.md`.
